# Design: Generalize meta-ppt-skills via Project Config

**Date:** 2026-03-04
**Status:** Approved

---

## Problem

All 17 module skills hardcode four categories of company-specific content:

1. **对齐概念** — 6 terminology rules specific to 数智生活商城 are duplicated verbatim in every skill
2. **Persona** — "通信行业竞争格局" assumes a telecom SOE context
3. **核心数据点** — specific figures from the 数智生活商城 PDF are embedded with no indication they're examples
4. **Skill trigger descriptions** — phrases like "重点突破五大场景" and "商城概览" hardcode document-specific wording into the trigger logic

This makes adaptation to other Chinese SOE business plans (healthcare, home-life, education, etc.) require editing all 17 files individually.

---

## Solution: Single `data/project-config.md` Config File

**Reuse scope:** Any Chinese SOE business plan PDF. Skills stay fixed; only the config changes when switching projects.

**Setup effort:** Edit one file (`data/project-config.md`). All 17 skills adapt automatically at runtime.

---

## New File: `data/project-config.md`

```markdown
# Project Config

## Platform
- **Name**: 数智生活商城
- **Company**: 中国移动
- **Industry**: 通信行业
- **Reference PDF**: data/PPT-数智生活商城商业计划书.pdf

## Persona Industry Context
你熟悉国资委管理体系、集团化运营逻辑和通信行业竞争格局。

## Alignment Terms (对齐概念)
- 平台正式名称: "数智生活商城"（不简写为"商城"，除非已在段落中定义）
- 虚拟货币: "数智豆"（不用"积分"替代）
- 三码体系: "身份码/权益码/支付码"（三码合一）
- 模式演进: BC模式 → BBC模式 → NBC模式（保留英文缩写）
- 治理层级: 集团总部 → 主体运营单位 → 省公司（三级结构）
- 战略框架: "一基+双核+两翼"（一基=商城，双核=数智豆+支付码，两翼=两类会员）
- 专业术语首次出现时，括号内给出简短释义
```

## Changes to 17 Module Skills

### 1. Persona — industry context becomes dynamic

**Before:**
```
你熟悉国资委管理体系、集团化运营逻辑和通信行业竞争格局。
```

**After:**
```
你熟悉国资委管理体系、集团化运营逻辑，以及本项目所在行业的竞争格局。
（行业背景详见 `data/project-config.md` → Persona Industry Context）
```

### 2. 对齐概念 — replace hardcoded rules with config reference

**Before:** 6 hardcoded 数智生活商城-specific terminology rules

**After:**
```markdown
## 对齐概念 / Concept Alignment (Universal Baseline)

**每次输出都必须应用此层，无论使用哪种风格。**

读取 `data/project-config.md` 中的 **Alignment Terms** 部分，
应用其中所有术语对齐规范。专业术语首次出现时，括号内给出简短释义。
```

### 3. 核心数据点 — labeled as reference example

**Before:** Data points presented as facts

**After:** Header line added:
```markdown
- **核心数据点** (数智生活商城参考示例 — 替换为您项目的数据):
```

The reference document line also becomes dynamic:
```markdown
- **参考文档**: 详见 `data/project-config.md` → Reference PDF
```

### 4. Skill trigger descriptions — generalized

Remove document-specific Chinese section titles from the `description:` frontmatter trigger line.

| Skill | Before | After |
|-------|--------|-------|
| key-scenarios | `重点突破五大场景 section` | `key scenarios or breakthrough opportunities section` |
| overview | `商城概览 section` | `overview or introduction section` |
| positioning | `商城定位 section` | `positioning or market positioning section` |
| strategy | `商城发展策略 section` | `development strategy section` |
| operations | `商城运营规划 section` | `operations planning section` |
| roadmap | `商城目标及实施路径 section` | `goals and implementation roadmap section` |
| ops-architecture | `商城平台运营架构 section` | `platform operations architecture section` |
| tech-architecture | `商城平台技术架构 section` | `platform technology architecture section` |
| business-model | `推进三种模式转变 section` | `business model transformation section` |

## Changes to `meta-ppt/SKILL.md`

### 1. Add Step 0 — read config before doing anything

```markdown
## Step 0: Load Project Config

Before showing the module picker, read `data/project-config.md`.
Use it to:
- Display the platform name in the module picker header
- Populate industry context in generated skill personas
- Populate Alignment Terms in generated skill 对齐概念 sections
```

### 2. Update SKILL TEMPLATE — emit config-reference version

The SKILL TEMPLATE's `对齐概念` block is updated to emit the dynamic version (config reference) rather than hardcoded rules. This ensures any future skill generated by `/meta-ppt` is automatically generic.

---

## What Does NOT Change

- Chinese section titles inside skill bodies (kept as gold-standard 数智生活商城 reference content)
- Module names and command names (`/key-scenarios`, `/overview`, etc.)
- Output format structure (标题 / 核心论断 / 数据支撑 / 行动建议 / 对齐概念)
- Style definitions (喊出口号 / 整体闭环 / 浓缩要义 / 总结亮点)
- News learning logic (`/learn-*` pattern)
- MODULE REGISTRY in meta-ppt (kept as reference for 数智生活商城 structure)
- All key data points (kept, labeled as examples)

---

## Adapting to a New Project

To use this suite for a healthcare or education SOE:

1. Drop the new PDF into `data/`
2. Edit `data/project-config.md`:
   - Update Platform name, Company, Industry, Reference PDF path
   - Update Persona Industry Context line
   - Replace the 6 Alignment Terms with your project's key terminology
3. Optionally: update `核心数据点` in each relevant module skill with your PDF's data
4. All 17 skills now produce output aligned to your project
